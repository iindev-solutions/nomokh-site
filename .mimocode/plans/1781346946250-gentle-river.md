# Plan: Nomokh — Nuxt 4 + Medusa.js Migration

## Current State
- Vanilla HTML/CSS/JS landing page (committed as reference on `new-v` branch)
- Design: dark/orange theme, grid layouts, GSAP animations
- Sections: Hero, About, Features, Catalog, CTA, Services, Contacts, Footer

## Tech Stack

| Layer | Tech | Purpose |
|-------|------|---------|
| Frontend | **Nuxt 4** | SSR, file-based routing, Vue 3 Composition API |
| E-commerce | **Medusa.js** | Products, orders, admin panel, inventory |
| Database | **PostgreSQL** | Required by Medusa |
| Analytics | **Plausible** (self-hosted) | Privacy-first, no cookies, 152-ФЗ compliant |
| Deployment | **Docker Compose** | Self-hosted on VPS |

## VPS Recommendation
- **Timeweb** (от 300₽/мес) — Российский, Docker поддержка
- **Hetzner** (от €4/мес) — Европейский, отличное соотношение цена/качество
- Docker Compose: medusa-server + postgres + plausible

## Product Management
- Товары хранятся в Medusa (PostgreSQL)
- Импорт из 1С/Excel → CSV → Medusa API
- Medusa Admin Panel — веб-интерфейс для добавления/редактирования товаров
- Загрузка фото, описание, цена, варианты (размер/цвет) — всё через UI

## Architecture

```
Browser → Nuxt 4 (SSR) → Medusa.js API → PostgreSQL
              ↓
     Product pages, catalog, cart, checkout
     Admin panel at /admin subdomain
```

## Implementation Phases

### Phase 1: Scaffold Nuxt 4 Project
- `npx nuxi@latest init nomokh-nuxt`
- Install dependencies: `@medusajs/medusa-js`, `@nuxt/image`
- Set up folder structure:
  ```
  nomokh-nuxt/
  ├── app/
  │   ├── components/     # Vue SFC components
  │   ├── layouts/        # Default layout with header/footer
  │   ├── pages/          # File-based routing
  │   ├── composables/    # useCart, useProducts, etc.
  │   └── assets/         # CSS, images
  ├── server/             # Nuxt server routes (proxy to Medusa)
  ├── public/             # Static assets (logo, favicon)
  ├── nuxt.config.ts
  └── docker-compose.yml  # Medusa + PostgreSQL
  ```

### Phase 2: Port Design System from Vanilla
Extract from current `styles.css`:
- CSS custom properties (tokens): `--color-orange`, `--color-dark`, etc.
- Font setup (Inter from Google Fonts)
- Layout utilities: `.l-wrapper`, grid patterns
- Shadow/border utility classes
- Component styles → Vue SFC `<style>` blocks

### Phase 3: Build Core Components
From vanilla HTML → Vue SFC:

| Vanilla Section | Vue Component | Notes |
|----------------|---------------|-------|
| Header | `AppHeader.vue` | Burger menu, nav, icons |
| Hero | `HeroSection.vue` | Grid layout, knife breakout |
| About | `AboutSection.vue` | Text + slider |
| Features | `FeaturesSection.vue` | 3-card grid |
| Catalog | `CatalogSection.vue` | 4-card grid (→ dynamic from Medusa) |
| CTA | `CtaSection.vue` | Grid 1fr 1fr |
| Services | `ServicesSection.vue` | 3-item list |
| Contacts | `ContactSection.vue` | Form + Yandex map |
| Footer | `AppFooter.vue` | Links, social, phone |

### Phase 4: Set Up Medusa.js
- Docker Compose: `medusa-server` + `postgres`
- Create product collections:
  - Ножи (Knives)
  - Термопосуда (Thermo cookware)
  - Одежда (Clothing)
  - Украшения (Jewelry)
- Admin panel setup at `/admin`
- Seed initial products

### Phase 5: Connect Nuxt ↔ Medusa
- `composables/useMedusa.ts` — Medusa client wrapper
- `composables/useProducts.ts` — fetch products
- `composables/useCart.ts` — cart state management
- Product listing page: `/catalog/[collection]`
- Product detail page: `/catalog/[collection]/[handle]`
- Cart page: `/cart`
- Checkout flow

### Phase 6: Dynamic Pages
- `/` — Home (all sections, catalog from Medusa)
- `/catalog` — All collections
- `/catalog/[collection]` — Collection listing
- `/catalog/[collection]/[handle]` — Product detail
- `/cart` — Shopping cart
- `/contacts` — Contact form

### Phase 7: SEO & Performance
- Nuxt `useHead()` for meta tags per page
- `@nuxt/image` for optimized images
- Medusa product data → OG tags
- Schema.org structured data for products
- Sitemap generation

### Phase 8: Deploy
- Docker Compose on VPS (Timeweb/Hetzner)
- Services: medusa-server, postgres, plausible, nuxt
- Nuxt: `output: 'server'` for SSR
- PostgreSQL: persistent volume
- Plausible: self-hosted analytics (no cookies, 152-ФЗ)
- Reverse proxy (Nginx) for SSL/domain
- Domain: nomokh.ru or similar

## Key Decisions

1. **Keep vanilla site as `/reference/` folder** in the Nuxt project for design comparison
2. **Port CSS tokens first** — ensures visual consistency
3. **Medusa handles products/orders**, Nuxt handles presentation
4. **No Nuxt Content** — Medusa is the single source of truth for products
5. **Pinia for cart state** — client-side cart with Medusa sync
