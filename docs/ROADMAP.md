# NOMOKH — roadmap до показа руководителю

## Что оставить в MVP

1. Premium-лендинг с понятной историей бренда.
2. Каталог с фильтрами: категория, цена, сталь, материал рукояти, сортировка.
3. Карточка товара: фото, легенда, характеристики, наличие, добавление в корзину.
4. Корзина и checkout-stub.
5. Импорт товаров из 1C CommerceML 2.
6. Аналитика после cookie consent.
7. Admin API без тяжелой админ-панели.

## Что убрать или отложить

- Аукцион — код оставить, но выключить флагом `NUXT_PUBLIC_ENABLE_AUCTION=false`.
- Мультиязычность — позже.
- Marketplace на нескольких продавцов — позже.
- Отдельный backend — позже, если будет нагрузка/worker.
- Полная админка — после первого согласия владельца.

## Простые задачи

1. Купить домен и направить DNS A-record на VPS.
2. Поднять VPS: Ubuntu, Docker, firewall.
3. Заполнить `.env`: secrets, домен, YooKassa, SMS, delivery.
4. Поднять `docker compose up -d --build`.
5. Проверить `/api/health`, главную, каталог, карточку, корзину.
6. Загрузить 10–20 реальных товаров.
7. Подключить реальные фото.
8. Включить Yandex Metrica ID и цели.
9. Подготовить privacy/offerta.
10. Показать владельцу сценарий: каталог → товар → корзина → заказ.

## Следующий этап после согласия

- YooKassa: договор, shopId, secret, return URL, webhook.
- CDEK/Почта РФ: расчет доставки.
- SMS.ru/WhatsApp: OTP и уведомления.
- S3-compatible storage для фото.
- Admin UI: товары, заказы, sync logs.
- Backups PostgreSQL в S3.
- E2E тесты checkout/payment.
- Auction v2: ставки, таймер, optimistic locking, уведомления.
